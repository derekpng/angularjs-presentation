AngularJS - What is it?

  • Developed in 2009 by Miško Hevery and Adam Abrons
  • Open-source web application framework
  • Maintained by Google and the community
  • Aimed at simplifying development of single-page applications
  • Built around the belief that declarative programming is best for building UI components, while imperative programming is excellent for expressing application logic
  • Does this by decoupling DOM manipulations from application logic

It's All About Decoupling

  • MVC is a way of programming that keeps the model, view, and controller separated
  • Model (M)—code that manages the data, logic, and rules of the application
  • View (V)—code that generates the output representation of information (i.e., what the user sees)
  • Controller (C)—code that accepts user input to affect the model and/or view

What's Wrong with Vanilla jQuery?

Nothing. It's like comparing apples to oranges. jQuery is intended to be a DOM manipulation library, and, as such, there is no strict separation of Model, View, and Controller code. This tends to make application code bases:

  • Larger—more lines of code
  • More difficult to read—due to the mixing of Model, View, and Controller code
  • More difficult to maintain—since it is not very modular

This is significant for large, complex applications.

jQuery—Implicit programming

Javascript



// This is an array storing the data
people = [
	{name: 'Santa Claus', age: 539, gender: 'male'},
	{name: 'Yeti', age: 18, gender: 'unknown'},
	{name: 'Fairy Godmother', age: 'unknown', gender: 'female'}
];
// This code generates the HTML table rows with the above data incorporated
var i, html = '';
for (i = 0; i < people.length; i++) {
	html += '<tr>';
	html += '<td>' + people[i].name + '</td>';
	html += '<td>' + people[i].age + '</td>';
	html += '<td>' + people[i].gender + '</td>';
	html += '</tr>';
}
$('#table-data').html(html); // Insert into html page

HTML

		<th>Name</th>

		<th>Gender</th>
	</tr>
	<tbody id="table-data">
	</tbody>
</table>

AngularJS—Declarative Programming

Javascript



// This is the model -- i.e., where the data is stored; any changes 
// to this data will automatically update the HTML (view)
$scope.people = [
	{name: 'Santa Claus', age: 539, gender: 'male'},
	{name: 'Yeti', age: 18, gender: 'unknown'},
	{name: 'Fairy Godmother', age: 'unknown', gender: 'female'}
];

HTML



<table>
	<tr>
		<th>Name</th>
		<th>Age</th>
		<th>Gender</th>
		</tr>
	<tbody>
		<tr ng-repeat="person in people">
			<td>{{person.name}}</td>
			<td>{{person.age}}</td>
			<td>{{person.gender}}</td>
		</tr>
	</tbody>
</table>

Say Goodbye to HTML Hacking

We often hack HTML in order to make it do something that is not included in the box. For example, if we want to make a tabbed panel:

  • One
  • Two
  • Three
Hi.
Hello.
Bonjour.

Non-ideal HTML



<div id="tabs">
  <ul>
    <li><a href="#fragment-1">One</a></li>
    <li><a href="#fragment-2">Two</a></li>
    <li><a href="#fragment-3">Three</a></li>
  </ul>
  <div id="fragment-1">
    Hi.
  </div>
  <div id="fragment-2">
    Hello.
  </div>
  <div id="fragment-3">
    Bonjour.
  </div>
</div>

This code is hard to read, because its intent/semantics is not clear.

Ideal HTML



<tabs>
  <pane title="One">
    Hi.
  </pane>
  <pane title="Two">
    Hello.
  </pane>
  <pane title="Three">
    Bonjour.
  </pane>
</tabs>

AngularJS allows you to make your own semantic, reusable HTML components. Declarative programming for the win!
